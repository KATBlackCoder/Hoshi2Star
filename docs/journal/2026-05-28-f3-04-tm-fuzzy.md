`similarity_score()` normalisé entre 0.0 et 1.0. 5 tests unitaires.

### Step 3 — lookup_fuzzy()
Scan complet en mémoire de toutes les entrées TM pour le `lang_pair`.
Normalise source avant comparaison (trim + lowercase, cohérent avec `hash_source`).
Score 1.0 → `match_type = "exact"`, sinon `"fuzzy"`. Tri décroissant, limite configurable.
6 tests unitaires couvrant cas limites (TM vide, dissimilaire, threshold, ordre).

### Step 4 — get_tm_suggestions command
Remplacement de `lookup_exact` par `lookup_fuzzy` (threshold 0.80, limit 5).

### Step 5 — TMPanel.tsx

`TmEntry[]` → `TmSuggestion[]`. Accès aux données via `suggestion.entry.*`.
Clés i18n `exact`, `fuzzy` ajoutées dans `en.json` et `fr.json`.

### Step 6 — generate_tmx + export_tm
`generate_tmx()` dans `tm.rs` : génère TMX 1.4 avec `std::fmt::Write`, helper `xml_escape()`.
`split_once('-')` utilisé à la place de `splitn(2, '-')` — correction clippy.

2 tests : structure TMX + échappement XML.

### Step 7 — Bouton Export TMX

`save()` de `@tauri-apps/plugin-dialog`, `useState` pour `isExporting`, toast sonner.

## Décisions prises

**Levenshtein manuel vs strsim** : implémentation ~30 lignes sur `char`s, 0 dépendance
externe. `strsim` n'aurait ajouté qu'une dépendance transitive pour ~20 lignes de code.
Choix cohérent avec le style du projet (minimalisme des dépendances).

**lookup_exact conservée** : utilisée dans `pipeline.rs` ligne 193 pour déduplication TM
avant envoi LLM (évite de re-traduire ce qui est déjà dans la TM). Ce n'est pas un doublon —
rôles différents : lookup_fuzzy pour l'UI, lookup_exact pour l'optimisation pipeline.

**Scan mémoire complet** : acceptable jusqu'à ~5k entrées (commentaire dans le code).
Pour les TMs plus grandes, un index trigram SQLite FTS est backlogué en F5.
Performance observée : non mesurée sur vrai jeu (TM vide en dev), mais O(n) sur des
chaînes de ~50 chars est rapide en Rust pour ~5k entrées (<5ms estimé).

**`project_id: Option<String>` retiré d'export_tm** : l'export filtré par projet est backlogué.
L'export global (toute la TM pour un lang_pair) couvre 100 % des cas d'usage beta.

## Problèmes rencontrés

- `splitn(2, '-')` → clippy `-D warnings` : `needless_splitn` et `manual_split_once`.
  Corrigé avec `split_once('-')` dans `tm.rs` et `project.rs`.
- `export_tm` introuvable dans `generate_handler!` → import manquant dans `lib.rs`.
  Ajout de `export_tm` dans la ligne `use commands::project::{...}`.
