## Bug 1 — Segments vides dans la grille (extractor.rs)

**Symptôme** : Map001.json et autres fichiers Map affichaient des lignes avec `source_text` vide dans la grille (ex : lignes 23, 26, 29…).

**Cause racine** : Le filtre d'extraction utilisait `!text.is_empty()` qui ne filtre pas les chaînes contenant uniquement des espaces (`"   "`, `"\t\n"`). Les maps RPG Maker utilisent parfois des lignes de dialogue vides/espace comme padding entre groupes de dialogue.

**Fix** :
- `src-tauri/src/engines/mv_mz/extractor.rs` — remplacement systématique (`replace_all`) de tous les checks `is_empty()` par `trim().is_empty()` sur les 4 patterns de variables : `text`, `name`, `title`, `currency`
- S'applique à toutes les fonctions d'extraction : `extract_event_list` (codes 101/401/102), `extract_simple_array`, `extract_skills`, `extract_states`, `extract_system`, `extract_common_events`

**Tests ajoutés** (+2 → 85 total) :
- `test_extract_skips_whitespace_only` — `""`, `"   "`, `"\t\n"` ignorés, seul `"有効なテキスト"` gardé
- `test_extract_simple_array_skips_whitespace` — `"  "` pour name ignoré, `""` pour profile ignoré, `"勇者"` pour nickname gardé

## Décisions prises

- **trim() côté extraction, pas côté DB** : le filtre whitespace-only est appliqué à l'extraction (avant insertion en DB), pas à la lecture. Évite de corrompre des segments déjà en DB avec des espaces significatifs (cas edge rare).
- **enabled: !!sourceText** : on n'exige plus `targetText !== null` pour lancer le QA check. Un segment non traduit (targetText = "") peut être vérifié — QA retourne score 100 si aucun placeholder manquant.
- **activeSegmentIdRef dans handleSave** : même pattern que les refs pour le listener LLM completed. Permet à `handleSave` (useCallback stable) de lire l'id actif sans se re-créer à chaque changement de sélection.
- **sonner plutôt que react-hot-toast** : shadcn v4 recommande sonner nativement. Compatible avec la config components.json existante.

## Bug 4 — Segments placeholder-only extraits (extractor.rs)

**Symptôme** : Map001.json et autres fichiers Map affichaient des segments contenant
uniquement des escape codes RPG Maker (ex : `\N[4]`, `\C[2]\N[4]`, `\.`) — rien à
traduire, mais extraits quand même, polluant la grille de ~10-20% de lignes inutiles.

**Cause racine** : Le filtre existant (`trim().is_empty()`) ne couvre que les chaînes
vides/whitespace. Une chaîne comme `\N[4]` n'est pas vide, donc passait le filtre.

**Fix** :
- `src-tauri/src/engines/mv_mz/extractor.rs` — ajout de `is_placeholder_only()` :
  tokenise le texte via `Tokenizer::tokenize(text, TokEngine::MvMz)`, retire tous les
  tokens `⟦ph_N⟧` du résultat, vérifie si le reste est vide/whitespace.
- `if tok.map.is_empty() { return false }` : si aucun placeholder détecté, le texte est
  du contenu pur — jamais skippé par ce filtre.
- Filtre ajouté sur les 3 codes dans `extract_event_list` : 101 (nom locuteur), 401
  (dialogue), 102 (choix).

**Note impl** : `Tokenizer::tokenize` remplace les placeholders par `⟦ph_N⟧` (pas par
`""`). Donc vérifier `.text.trim().is_empty()` ne suffit pas — il faut retirer les
tokens du résultat pour trouver le contenu réel.

**Tests ajoutés** (+4 → 89 total) :
- `test_extract_skips_placeholder_only_segment` — `\N[4]` seul → skippé, texte valide → gardé
- `test_extract_keeps_placeholder_with_text` — `\N[1] 反応なし…` → gardé
- `test_extract_skips_multiple_placeholders_no_text` — `\C[2]\N[4]` → skippé
- `test_extract_keeps_text_with_intercalated_placeholder` — `捕えた！\V[5]水深１５m` → gardé

## Bug 5 — `\n[n]` lowercase non reconnu par le tokenizer

**Symptôme** : Après le Bug 4, des segments `\n[1]` restaient visibles dans la grille —
le filtre `is_placeholder_only` ne les détectait pas. Screenshot post-fix montre encore
des lignes 249, 251 = `\n[1]`.

**Cause racine** : `RE_MVMZ` utilisait `[VNPCI]` (majuscules seulement). La spec
officielle RPG Maker utilise les majuscules, mais ce jeu utilise des minuscules
(`\n[n]`, `\c[n]`) — variantes répandues dans les community plugins.

**Fix** :
- `src-tauri/src/llm/tokenizer.rs` — `RE_MVMZ` et `RE_MZONLY` :
  - Groupe A : `[VNPCI]` → `[VNPCIvnpci]`
  - Groupe C : `(?:PX|PY|FS)` → `(?:PX|PY|FS|px|py|fs)`
- Round-trip préservé : `\n[1]` tokenisé → `⟦ph_0⟧` → restauré en `\n[1]` (casse conservée)

**Tests ajoutés** (+1 → 90 total) :
- `test_lowercase_codes_tokenized` (tokenizer) — `\n[1]` seul, `\n[1] 反応なし…`, round-trip `\c[3]\n[1] テスト`

**Tests mis à jour** (extractor) :
- `test_extract_skips_placeholder_only_segment` — utilise maintenant `\n[1]` + `\N[4]`
- `test_extract_keeps_placeholder_with_text` — utilise `\n[1] 反応なし…` (cas réel)
- `test_extract_skips_multiple_placeholders_no_text` — utilise `\c[2]\n[4]` (lowercase)

**Docs mises à jour** :
- `docs/engines/mv-mz-placeholders.md` — regex, tableau, tests unitaires, historique
