## Décisions prises

**Step 3 — méthode `chat()` sur le trait `LlmProvider`** : le plan laissait le choix entre ajouter au trait ou garder autonome. Ajout au trait retenu pour la testabilité avec httpmock (pas besoin de mock HTTP séparé). La méthode est générique (system + user → raw string) et réutilisable.

**Step 8 — highlight réalisé (non reporté)** : l'imbrication `HighlightedSource` (dans App.tsx) + glossaire aurait créé une dépendance awkward. Solution retenue : `SourceCell` dans `columns.tsx` implémente les deux highlights en une seule passe (glossaire split d'abord, placeholders dans chaque segment de texte brut). Algorithme clean, 0 overlap problématique en pratique (les placeholders `\V[n]` n'apparaissent pas dans les noms propres).

**Step 4 — project_id résolu depuis segments** : `translate_segments` prend des segment IDs ou un file_id. Résolution du `project_id` via JOIN `segments → source_files` à l'intérieur du `tokio::spawn`. Une requête supplémentaire mais acceptable (exécutée une fois par traduction).
