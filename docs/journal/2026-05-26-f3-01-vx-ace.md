## Décisions prises

- **API marshal-rs** : `load_utf8(&[u8], None) -> Result<marshal_rs::Value, _>` et `dump(marshal_rs::Value, None) -> Vec<u8>` — `marshal_rs::Value` ≠ `serde_json::Value`, conversion bidirectionnelle via `Into`
- **Round-trip test** : `load_utf8(dump(load_utf8(bytes))) == load_utf8(bytes)` — les bytes peuvent différer (encodage), pas les Value décodées
- **Ordre détection** : MV/MZ toujours vérifié en premier (System.json + gameTitle), puis VX Ace (System.rvdata2)
- **Types fichiers** : préfixe `"vx_*"` en DB pour distinguer VX Ace des types MV/MZ sans charger le moteur du projet
- **is_placeholder_only** : réutilisé directement depuis MV/MZ — VX Ace utilise les mêmes escape codes

## Problèmes rencontrés

- **gxhash compile error** : marshal-rs 2.0.1 → gxhash → nécessite AES+SSE2. Résolu : `.cargo/config.toml` avec `rustflags = ["-C", "target-cpu=native"]`
- **API mismatch** : documentation initiale incorrecte. Résolu : test d'investigation au Step 2 qui a révélé l'API réelle avant d'implémenter le reste
