### Step 4 — Extracteur .mps via wolfrpg-map-parser (commit 7770356)

#### API wolfrpg-map-parser 0.6.0 — découvertes

| Element | Détail |
|---------|--------|
| Parser principal | `Map::parse(bytes: &[u8]) -> Map` — **panics** si bytes invalides |
| Events | `map.events() -> &Vec<Event>` |
| Pages | `event.pages() -> &Vec<Page>` |
| Commands | `page.commands() -> &Vec<Command>` |
| ShowMessage | `Command::ShowMessage(ShowTextCommand)` — `.text() -> &str` |
| ShowChoice | `Command::ShowChoice(ShowChoiceCommand)` — `.choices() -> &Vec<String>` |
| Encoding | La crate gère **elle-même** le décodage SJIS via `encoding_rs::SHIFT_JIS` dans `byte_utils::as_string`. Pas besoin de pré-décoder les bytes avant de passer à `Map::parse`. |
| Signatures | ShowMessage = `0x01650000`, ShowChoice = `0x02660000` (voir `command/signature.rs`) |

**Choices supportées** : oui, via `Command::ShowChoice`. Les clés choices ont le format
`MapData/{map}/events/{e}/pages/{p}/{cmd}/choices/{c}`.

#### Implémentation

`extract_map_segments(map_name, bytes, version) -> Result<Vec<WolfSegment>, ExtractorError>` :
- `std::panic::catch_unwind` pour capturer les panics de `Map::parse`
- Itère events → pages → commands
- `Command::ShowMessage` → 1 segment `MapMessage`
- `Command::ShowChoice` → 1 segment par choice
- Filtre vide + placeholder-only

`is_wolf_placeholder_only(text)` — **bug corrigé en cours de session** :
- Erreur initiale : `tokenized.text.trim().is_empty()` — incorrect car `.text` contient `⟦ph_0⟧` pas `""`
- Correction : même pattern que MV/MZ — retirer les tokens via `fold`, vérifier si `bare.trim().is_empty()`

#### Bytes .mps synthétiques pour les tests

Format des composants :
- String SJIS : `u32_le(length)` + SJIS bytes + `0x00` null (length inclut le null)
- ShowMessage command : 4 bytes sig BE + 1 pad + 2 unknown + string + 1 end
- Page : PAGE_SIGNATURE + icon + props + 36B conditions + 6B move + move_count + cmd_count + cmds + 4B unknown2 + 3B extensions + 0x7a
- Event : EVENT_SIGNATURE + 1 pad + id + name + x + y + page_count + unknown + pages + 0x70
- Map : MAP_SIGNATURE + 5B unknown + skippable + tileset + w + h + event_count + layers + events + 0x66

Fichiers réels utilisés pour les tests : **aucun** — 100% synthétiques.

## Corrections documentées

### Bug is_wolf_placeholder_only

**Erreur** : vérifier `tokenized.text.trim().is_empty()` — le `.text` contient les tokens
opaques `⟦ph_0⟧`, donc n'est jamais vide même si le segment est 100% placeholder.

**Correction** : retirer les tokens du texte avec fold (pattern identique à MV/MZ) avant de tester.

```rust
let bare = tok.map.keys().fold(tok.text.clone(), |s, k| s.replace(k.as_str(), ""));
bare.trim().is_empty()
```

**Règle** : toujours utiliser le pattern fold pour `is_placeholder_only`, pas `tokenized.text.trim()`.
