# ADR-001 : SQLite via sqlx async

## Statut
Accepté — 2026-05-24

## Contexte
La DB locale stocke TM, glossaire, projets et segments.
TanStack Table peut déclencher pagination + filtres en concurrent.
Tauri expose un plugin SQL officiel mais il donne accès à la DB depuis le JS.

## Décision
sqlx 0.8 avec feature sqlite + runtime-tokio.

## Alternatives rejetées
- rusqlite : synchrone, bloque le thread IPC Tauri sous charge
- tauri-plugin-sql : expose la DB au JS, frontière Rust/TS cassée

## Conséquences
+ Requêtes async non-bloquantes
+ DB inaccessible depuis le frontend (sécurité)
- Migrations gérées manuellement via sqlx-cli
