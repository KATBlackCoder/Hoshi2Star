# ADR-003 : Translation Memory globale à l'installation

## Statut
Accepté — 2026-05-24

## Contexte
Translator++ (concurrent principal) a une TM par projet.
Si un traducteur a déjà traduit "魔法使い → mage" dans 10 jeux,
cette information est perdue à chaque nouveau projet.

## Décision
La TM est globale à l'installation de Hoshi2Star, pas par projet.
Chaque segment validé alimente la TM commune.
Le lookup fuzzy cross-projet est le différenciateur clé vs Translator++.

## Alternatives rejetées
- TM par projet : silo, perd la valeur accumulée
- TM cloud partagée entre users : complexité infra, vie privée (backlog F5)

## Conséquences
+ Suggestions s'améliorent avec le temps et le nombre de projets
+ Export TMX standard (compatibilité OmegaT/Trados)
- La TM grossit avec le temps (prévoir purge optionnelle)
