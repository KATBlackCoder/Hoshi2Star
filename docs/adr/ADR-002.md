# ADR-002 : Tokenisation des placeholders côté Rust avant LLM

## Statut
Accepté — 2026-05-24

## Contexte
Les jeux RPG Maker contiennent des codes d'échappement dans les textes :
\V[n], \C[n], \N[n], \I[n], etc.
Les LLMs (Ollama, OpenAI, DeepSeek) les cassent ou les suppriment
systématiquement quand ils apparaissent dans les prompts bruts.

## Décision
Avant tout envoi LLM, le tokenizer Rust remplace chaque placeholder
par un UUID opaque (ex: ⟦ph_001⟧). Après réponse, restauration et
validation — si un UUID est manquant ou dupliqué, le segment est rejeté
et renvoyé.

## Alternatives rejetées
- JSONObjectCloaking (approche Translator++) : fragile, dépend du format
- Laisser les placeholders bruts : taux d'erreur trop élevé en pratique

## Conséquences
+ Zéro placeholder cassé en sortie de pipeline
+ Validation automatique avant réinjection
- Légère complexité supplémentaire dans le pipeline LLM

## Patterns à tokeniser (extrait du rapport deep research)

Avec argument numérique :
- \V[n]  \N[n]  \P[n]  \C[n]  \I[n]
- \PX[n]  \PY[n]  \FS[n]  (MZ uniquement)

Sans argument :
- \G  \\  \$  \.  \|  \!  \>  \<  \^  \{  \}

Substitutions dynamiques (à préserver) :
- [%n]  (MV uniquement — Terms > Messages)
- %n    (MZ uniquement — Terms Settings > Messages)
