# ADR-005 : lib.rs comme entrée application

## Statut
Accepté — 2026-05-24

## Contexte
Tauri v2 supporte les builds mobiles (iOS, Android).
Les builds mobiles Tauri requièrent que la logique principale
soit dans lib.rs et non main.rs — main.rs ne peut pas être
le point d'entrée sur mobile.

## Décision
lib.rs contient tauri::Builder et toute la logique d'initialisation.
main.rs est minimal : il appelle uniquement lib::run().

## Alternatives rejetées
- Tout dans main.rs : casse les builds mobiles futurs

## Conséquences
+ Compatible builds desktop + mobile sans refactoring futur
+ Pattern standard recommandé par la doc Tauri v2 officielle
- Légère indirection main.rs → lib.rs (négligeable)
